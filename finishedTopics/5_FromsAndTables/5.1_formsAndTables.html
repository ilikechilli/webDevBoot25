Forms and Tables

Tables

Html allows the creation of tables. Theses tables should be used to display tabular data and not to structure a web page since there are much better tools for that.

Tables exist within a <table> element and can take several children elements, some of which are required for the table creation

  • TR = A row of data
  • TD = A single cell of data
  • T
  • thead
  • tfoot

The following is an example of a table listing large birds

Type of bird Average weight kg (lbs) Max weight Flighted
Ostrich 104kg (230 lbs) 156.4 kg no
Somali Ostrich 90 kg (200 lbs) 130 kg no
Wild turkey 13.5 39 yes

The table above is semantically correct but not yet styled. It is also possible to have several columns or sows span differently using the <colspan> and <rowspan> elements.

Forms

Forms are all over the place on the web as all kinds of inputs. Forms are initialized with the <form> element and usually contain <input> and <label> elements. Common input types are:

  • text
  • button
  • radio
  • checkboxes
  • password
  • color
  • number
  • time
  • date

The form element will also specify what to do with the data. The parent form element is where we specify what data should be sent where.

Forms have many attributes but the most important ones are "action" and "method". The action specifies where the user needs to go once submitting the form and the method specifies what needs to happen (GET, POST...) In the form demo below, the action is "/chickens".

Labels may seem like simple elements but they can be very powerful but only if we first introduce two new attributes to the input tag, namely the "name" and "id" attributes.

The id attribute assigns a specific "behind-the-scenes" id to an input. Now, when looking at the label element, we will often give it a "for" attribute which will point to the name of the input that the label is for. When to elements are associated through the "id" and "for" attributes, it also makes the label clickable to control the input. The name attribute is used later when sending information over the web. But we'll get to that in a short while

Buttons

The button element represents a clickable button. If the button is present within a form it will submit the form by default. Even if multiple buttons are present, they will all submit the form in which they are present. Moving a button outside of a form, it won't do anything. If there is a button inside a form that should not submit the form, a type attribute should be added to the "button"


The name attribute

We've seen that we can add labels to inputs and that we can use buttons to submit forms. But what happens when a form is submitted anyway? For that, we'll first need to introduce the "name" attribute which is given to an <input> tag

Once submitting, the inputs are added to the body of the request (See the URL in the browser).In the example form below, if you choose to fill in the the form and check the checkbox and finally submit, you will see that the URL changes to the "action" alongside the names behind it. Checkboxes, Radio buttons and Selects





Radio buttons take advantage of both the name and id attribute. All radio buttons with the same name attribute will make it so that only a single button can be selected for that name (they are grouped). So if multiple radio buttons have the same name only one can be checked. If you then check another radio button with the same name attribute, the previously checked button will no longer be checked. In order to make sure that the radio button actually submits something it should also have an ID that is linked to a label that has a value. The value can be added as a separate attribute but will not be displayed to a user.

Select inputs have a slightly different way of operation. It does not take an input tag but the <select> tag which can include any number of <option> tags within it. Each option can have a value which is then sent through when submitting. The selected value will be associated to the name (not the idea) or the select tag.

Range and Text-Area

Range inputs provide a slider that a user can use to select a previously determined range of values using the "min" and "max" attributes. These attributes are also useful when using a number input when you want to constrain the number that someone can send through.



textareas are not an input but have their own tag <textarea> but they do function as an input. The difference with a simple text input is that we can control the size of the text input box! To set the size, we can use "rows" and "cols" attributes for the text area.



Html form validation (Built in validations)

Sometimes there are certain requirements for inputs. For example, is an email valid? Does a password contain certain character types? Many validations are done server side using javascript but there are also built in validations in the browser.

Some validations can include

  • required = This ensures that an input is provided before submitting.
  • minlength = the minimum length of text
  • maxlength = the maximum length of text
