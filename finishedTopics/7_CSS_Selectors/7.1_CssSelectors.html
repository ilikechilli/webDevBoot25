CSS Selectors

This page will cover a wide range of different CSS selectors and a short section about CSS specificity. The most important selectors that will be discussed are as follows:

  • Element selectors
  • Class Selectors (.)
  • ID selectors (#)
  • Descendent selectors

Other selectors that are worth knowing about are:

  • Adjacent selectors
  • Direct descendent selectors
  • Attribute selectors
  • Pseudo elements
  • Pseudo classes

Universal and Element selectors

The universal selector is "*" and it selects EVERYTHING

Element selectors select every html element of a certain type. For example, all paragraphs are selected using "p"

Selector lists can be made to select multiple types of tags and are accomplished by using a comma between selectors. For example: "h1, h2 {}" will select all h1 and h2 headers and a style can be applied between the curly braces for both header types.

The ID selector

Elements with a specific ID can be selected by using the # symbol followed by the ID. This specific paragraph had an ID of "IDSelectorParagraph" and has a specific style applied to it that makes the text color darkslateblue. It will only apply to this paragraph because only this paragraph has this ID. The selector in the CSS file is "#IDSelectorParagraph {color: darkslateblue}".

The Class Selector

Class selectors like ID's can be added to our markup (html) with the biggest difference that is can be applied to multiple elements (A class of elements). Classes can be targetted in CSS using a dot before the class name. For example, if I make a class in my html that is named "redText", I can target it in CSS using ".redText{}"

Let's say we create a CSS class like redText, we can apply it to this by wrapping 'this' in a span and adding the class attribute to it with the value redText and then add it in the CSS. This should then show up as red.

Descendent selectors

Descendent selectors use a selector followed by a space and another selector that is a descendent of the first selector. Note that this is not the same as a child (or direct child) selector. Descendent selectors do not have to be directly within the ancestor selector.

Adjacent Selector and Direct-Descendent Selector

Adjacent selectors both reside within a common parent selector and follow eachother directly. Imagine I have a div containing an h3 header and a paragraph right after it. These are Adjacent and can be selected by using "h3 + p {}"

Direct child selectors are a bit like Descendent selectors but the child element needs to reside within the parent element directly. It can be targeted using the ">" symbol like this "parent > child {}"

The attribute selector

The attribute selector allows the selection of elements based on a specific attribute. This is particularly useful with the <input> element because besed on the "type" attribute, the element can have very different functions. So using the attribute selector, we can select a single type of input element to style. For example: "input[type="text"]{}"" will target only the text input elements.

Pseudo Classes

They are keywords added to the end of a selector that indicate a specific state of the element. They all start with a colon and one of the most commonly used ones is :hover.

Other examples of anchor tags are:

  • element:hover hovering over this element with your mouse will turn it blue
  • element:active While clicking on this element you can change its style. Useful for buttons when being clicked. Click this paragraph to see it in action.
  • element:checed This is useful for checkboxes or radio buttons that are checked.
  • element:nth-of-type this selects elements among a group of sibling elements based on their (numbered position) Notice how in this list there are a few entries that are differently styled? That is nth-of-type at work!
  • ... and many more. See MDN docs for more information on Pseudo Classes

A bit more on the nth-of-type pseudo selector. This selector takes an argument. Simply providing a number as an argument will target the element that is numbered at that position in the list. But we can also provide a repeating sequence by providing a number followed by "n". In that way, passing 2n would target every other element. Providing 3n will target every third element in the list and so on. You can also provide the keywords even or odd or you can tartget odd elements with (2n - 1).

Pseudo Elements

Pseudo Elements are keywords added to a selector that lets you modify a certain part of the selected elements. They are recognized by the double colon "selector::pseudoselector". There are not a lot of pseudo elements but some include:

  • ::first-letter which targets the first letter so that it can be styled separate.
  • ::first-line
  • ::selection (targets highlighted elements)
  • ::before
  • ::after

The CSS Cascade

The cascade refers to the order that styles are applied. If an element is targeted by more than one CSS rule, the last rule to be typed will be applied and the previous rules will be ignored, or more accurately; overwritten. The order also matters if there are more than one stylesheets.

Css specificity

Order matters but the "type of CSS also matters" Inline CSS has precedence over Styled-tag CSS and STyled-tag CSS has precedence over an external CSS file!

If there is a conflict of more than one style that are competing for element style the precedence is as followed:

  1. More element selectors
  2. ID beats class
  3. Class beats elements

Inline Styles and !Important

Inline styles are more specific than ID's in external stylesheets. Inline styles are not recommended but are a valid way of styling nonetheless.

The important rule is generally not recommended to be used and it is the most specific rule possible. It will override everything else!

CSS inheritance

Some CSS properties will be inherited by child elements even if those styles are not set to the child explicitely

For example, if the body is given a color property of red, then all the text in the paragraphs in the body are going to be red because it is inherited from the body. Certain elements can inherit certain styles from the closest parent where the style is set.
