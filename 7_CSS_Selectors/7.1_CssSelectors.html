CSS Selectors

This page will cover a wide range of different CSS selectors and a short section about CSS specificity. The most important selectors that will be discussed are as follows:

  • Element selectors
  • Class Selectors (.)
  • ID selectors (#)
  • Descendent selectors

Other selectors that are worth knowing about are:

  • Adjacent selectors
  • Direct descendent selectors
  • Attribute selectors
  • Pseudo elements
  • Pseudo classes

Universal and Element selectors

The universal selector is "*" and it selects EVERYTHING

Element selectors select every html element of a certain type. For example, all paragraphs are selected using "p"

Selector lists can be made to select multiple types of tags and are accomplished by using a comma between selectors. For example: "h1, h2 {}" will select all h1 and h2 headers and a style can be applied between the curly braces for both header types.

The ID selector

Elements with a specific ID can be selected by using the # symbol followed by the ID. This specific paragraph had an ID of "IDSelectorParagraph" and has a specific style applied to it that makes the text color darkslateblue. It will only apply to this paragraph because only this paragraph has this ID. The selector in the CSS file is "#IDSelectorParagraph {color: darkslateblue}".

The Class Selector

Class selectors like ID's can be added to our markup (html) with the biggest difference that is can be applied to multiple elements (A class of elements). Classes can be targetted in CSS using a dot before the class name. For example, if I make a class in my html that is named "redText", I can target it in CSS using ".redText{}"

Let's say we create a CSS class like redText, we can apply it to this by wrapping 'this' in a span and adding the class attribute to it with the value redText and then add it in the CSS. This should then show up as red.

Descendent selectors

Descendent selectors use a selector followed by a space and another selector that is a descendent of the first selector. Note that this is not the same as a child (or direct child) selector. Descendent selectors do not have to be directly within the ancestor selector.

Adjacent Selector and Direct-Descendent Selector

Adjacent selectors both reside within a common parent selector and follow eachother directly. Imagine I have a div containing an h3 header and a paragraph right after it. These are Adjacent and can be selected by using "h3 + p {}"

Direct child selectors are a bit like Descendent selectors but the child element needs to reside within the parent element directly. It can be targeted using the ">" symbol like this "parent > child {}"
