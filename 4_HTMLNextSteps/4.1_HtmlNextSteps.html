Inline versus block elements

Inline elements will not require their own line and can be placed 'in line' with other elements on the page

Block elements will populate their own new line.

Whether an element is inline or block is predetermined and each element will be one of those two but this can be changed using CSS

Div elements: The content division element

This is a generic container to hold things and group things together. A div is a block style element.

Span

Just like the div, a span element is a generic element that can be wrapped around certain content. A span is an inline element. You can use classes to target a span and give it unique styling.

HTML Entities

These are special codes or sequences that represent very specific character such a copyright symbol ©

HTML entities are very useful when attempting to type reserved characters in HTML such as the "Less than" or "Greather than" characters

  • < is written in html as "&lt;"
  • > is written in htmls as "&gt;"

Notice that the ampersand and semicolon are also special characters themselves so just to display them the way that they are displayed here could be done by using Entities

  • & can be written as "&amp;"which is confusing because it includes an ampersand to encode it.
  • ; can be written as "&semi;"

Semantic markup

Semantic = related to meaning

Semantic Markup = meaningful markup

Some websites use a multitude of div elements but these can be overwhelming and non descriptive. newer HTML allows the use of semantically meaningfull elements that function the same way as a div but are more descriptive of what their function should be.

Some semantically relevant elements that function as a div are:

  • nav
  • header
  • footer
  • main
  • section
  • article
  • aside
  • summary
  • ...
