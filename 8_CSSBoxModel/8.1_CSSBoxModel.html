CSS Box Model

The properties of the CSS box are:

  • margin
  • padding
  • border
  • widht and height

Closely related topics are:

  • Display Property
  • Units: Percentage, EM and REM
  • Border radius

Everything in CSS can be considered a box defined by margin, padding, border, width, height and its display property

Width and height

Width and height control the inner content area (exception for border box...) of an element. Most (if not all) Block type elements will have a width that spans the width of the parent element (or the page). This can be overwritten by explicitely setting the width for that element using CSS. For example, a standard paragraph element will span the width of the page but can be made shorter if required.

If the width and height parameters are set too low, the content may overflow and cause strange effects.

Other properties will also only be applied to the content area such as the background color.

Border

The border is the border around an element.

Borders are fairly self-explanatory but they can be used to very good effect to make things clear or to point things out on a web-page.

Borders can be modified in thickness, color, style (dotted, dashed, full...) using the respective properties.

By default, an element size does not include the border but there is a property called box-sizing set to "border-box" which will cause the size of the element to go from border to border.

Padding

Padding is set between the content and the border of an element.

Margin

Margin is like padding but on the outside of the border... Nuff Said!

The Display Property

We've discussed inline and block style elements briefly in a previous section but there is more to be said about these property.

The first is about inline elements. If we try to apply a width that is wider than the content to an inline element, it will not apply because of the fact that an inline element will only fill the width that is required by the content! However, padding will be respected but it will not necessarily push other content away. Instead it may overlap here and there. The same goes for margin. It is respected but only in some cases. Introducing the "Inline-block" type display type! This solves issues encountered by inline elements with regard to sizing. Inline-block elements will sit inline but can be sized as required!

Display: none

This hides the element and can be useful later when we get to javascript!

CSS Units revisited

Relative units

Pixels are an absolute unit

Percentages

Setting the width of 50 sets the width of an element to 50% of the width of the parent element! So it is not the width of the whole page as it is displayed. Keep this in mind because nesting too much of this can cause weird stuff to happen.

EMs and REMs

EM

EM represent the relative size compared to the parent element. EM's are used in two contexts! For font sizes, 1em equals the font size of the parent. For other properties 1em is equal to the computed font size of the element itself.

Let's illustrate this with an example below. Where a paragraph sits within a div. I will set the font size of the div to 20 pixels and the font size of the paragraph to 2em. Since the paragraph is a child of the div, its font size will be 20 pixels or twice the size of the parent. But I have also set the margin of the paragraph to 1em. This margin size is not based on the the size of the parent but on the font size of the paragraph itself. So in this case 1em for the margin also represents 40 pixels.

My font size is 2em and my margin is 1 em but they are both 40 pixels because the font size for my parent element is 10 pixels.

REMs

REM is based on the font size of the root element and is very useful to avoid snowballing sizing effects when nesting multiple elements
